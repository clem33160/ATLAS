# SANDBOX CYCLE PROTOCOL

## Cycle 1 — Observer
Lire canon, pack plomberie, sandbox, tests, rapports.

## Cycle 2 — Détecter faiblesse
Identifier manque vérifiable (test, scénario, scoring, conformité, mémoire).

## Cycle 3 — Créer mission Codex
Créer mission contractuelle: objectif, périmètre, interdits, tests, acceptation/refus.

## Cycle 4 — Codex propose/code
Coder uniquement dans le périmètre autorisé.

## Cycle 5 — Atlas teste
Lancer tests structure, métier, conformité, sécurité, anti-chaos, anti-régression.

## Cycle 6 — Atlas compare
Comparer candidat vs état précédent/canon.

## Cycle 7 — Atlas accepte/refuse
Accepter seulement si meilleur ou égal partout sur dimensions critiques.

## Cycle 8 — Atlas journalise
Écrire cycle_log, scorecard, raisons de décision.

## Cycle 9 — Atlas décide prochaine mission
Écrire next_mission_recommendation.

## Cycle 10 — Stop intelligent
Arrêter si plateau, danger, budget faible, incertitude forte, tests cassés, mutation trop large.
